SUBSCRIPCIÓN & CANCELAR SUBSCRIPCIÓN

Como se sabe los observables son objetos que pueden manipular stream de datos. Pero para acceder a dicho stream de datos se tiene que hacer uso de una subscripción a dicho stream, para poder estar escuchando constantemente las actualizaciones de los datos que emite el observador.

Cabe resaltar que la mala implementación de los observadores puede llevar a que haya fugas de memoria, y en caso de hacer uso de multiples observadores simultaneamente, según que proceso puede generar perdidas de rendimiento dentro de la aplicación, ya que cada instancia del observador que tengamos va tener un proceso paralelo pendiente de un stream de datos constantemente.

Por ende asi como existe un metodo de subscripción a un observador, tambien se debe cancelar esta subscripción una ves ya no se necesita estar al pendiente del stream de datos o que ya no se tiene que renderizar el componente en el cual esta implementado dicho observador.

SUBSCRIPTION.

Suponiendo que se tiene el siguiente observable: 

          
            this.interval$ = new Observable < number  >( suscriber => {{'{'}}
              let i = 0;
              const interval = setInterval( () => {{'{'}}
                suscriber.next(i);
                i++;
              {{'}'}}, 1000);
            {{'}'}});

Para suscribirse se podria hacer de la siguiente manera, pero si se analiza el codigo más a profundidad se puede llegar a la conclusión que no hay una manera de cerrar ese stream de datos y va a estar constantemente emitiendo un valor e imprimiendolo en consola. 

          
            this.interval$.subscribe( console.log )

Una mejor manera de hacer el paso anterior es declarando una variable de tipo Subscription la cual guardara la referencia de la subscripción y permitira que esta sea cancelada en cualquier momento a travez del metodo unsubscribe

          
            public subscription: Subscription ;
    
            subscription = this.interval$.subscribe( console.log );

UNSUBSCRIPTION.

Para cerrar el stream de datos solo basta con tomas la variable en la cual esta la referencia a la subscripción y ejecutar el metodo unsubscribe esto inmediatamente cerrara el stream de datos al cual se estaba suscrito. Pero hay que tener cuidado cuando de ntro del observador se estan ejecutando procesos que necesitan ser cerrados para evitar fugas de memoria, en este caso el setInterval si no se cierra de algun modo va a seguir ejecutandose en segundo plano aunque ya se allá cancelado la subscripción al stream 

          
            this.subscription.unsubscribe();

TEARDOWNLOGIC.

Para el problema planteado anteriormente al ejecutar el unsubscribe se tiene el teardown el cual puede ser un callback que se ejecuta al momento de realizar la cancelación de la subscripción, en el cual se pueden cerrar todos los procesos internos que haga el observable que puedan quedar ejecutandose en segundo plano. Por lo tanto el codígo quedaria de la siguiente manera: 

          
            this.interval$ = new Observable< number >( suscriber => {{'{'}}
              let i = 0;
              // interval: Referencia al intervalo en ejecución
              const interval = setInterval( () => {{'{'}}
                suscriber.next(i);
                i++;
              {{'}'}}, 1000);
              
              // teardownlogic, se ejecuta al momento de cancelar la suscripción y elimina el intervalo que se estaba ejecutando
              return () => {{'{'}}
                clearInterval(interval);
              {{'}'}}
            {{'}'}});

IMPORTANTE:

  • Se debe tener cuidado al realizar una subscripción, si se ejecutan varios ordenes de subscripción en un corto latso de tiempo, y no se guarda referencia a cada una de estas subscripciones, puede quedar alguna fuga de memoria.
  • Si se cancela una subscripción y se vuelve a subscribir, el flujo de datos no empieza donde quedo en la pausa si no que vuelve e inicia desde el principio ya que es una nueva instancia.
  • Dentro de la suscripción se tiene el metodo add() el cual permite agregar un teardown (callback) que se ejecutara al momento de realizarse la cancelación de la subscripción. Tambien se tiene el metodo remove() que permite quitar un teardown agregado previamente, si se agrega el mismo teardown multiples veces con add() se debe llamar la misma cantidad de veces el remove().
  • En la suscripción se tiene el atributo closed el cual indica si una suscripción ya fue cerrada
